Project Introduction

Immunization Information Systems (IIS) receive immunization data from Electronic Health Record (EHR) systems and must ensure the data meets high standards for data quality. As IIS shift from direct user data entry to receiving data from EHR systems, the importance of data quality processes increases. The DQA is an open source software, created by the immunization community, to measure and ensure data quality of incoming and received data.

Software Use

The DQA is a Java web application, commonly deployed with Apache Tomcat, and connected to a database such as Oracle, Postgres or MySQL. The DQA accepts data via real-time POST interface or by batched files and produces a data quality report. The data quality report summarizes the quality of data in a set of weekly real-time submissions or a batch of data submitted in a file. IIS use the DQA in several ways:

  • Assist in manual review of data to be imported.
  • An auxiliary reporting system in the import process.
  • A fully integrated step in the import process.

Technical Details

Technology Java, Hibernate, Wicket
Database Oracle, Postgres, MySQL, or others
Interface HL7 v2.x, real-time and batch
Deployment Java Web Application on server or desktop
License Apache 2, open source and business friendly
